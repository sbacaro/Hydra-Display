# Hydra Display v0.3.0

**Color, brightness, and sharper Picture in Picture.** This release adds per-display
software dimming and white-point control, richer PIP windows, a built-in diagnostics
export, and integrity-verified auto-updates.

## ✨ Highlights

- **Software dimming & color temperature.** Set brightness and a warm↔cool white point
  for *any* display — physical or virtual — using the public CoreGraphics gamma API. No
  DDC, no extra hardware. Controls live on each Overview card (the sun button) and in the
  virtual-display detail pane. Settings persist per display, re-apply after sleep/wake,
  and revert to normal when you quit.
- **Better Picture in Picture.** Each floating window gains a hover-revealed strip to
  adjust **opacity** and toggle **click-through** (the pointer passes through to whatever's
  behind). Open as many PIP windows as you like; a new **Picture in Picture** section in
  the menu bar lists them all and is the reliable way to turn click-through back off or
  close a window. The pointer is no longer drawn into the feed.
- **Diagnostics export.** Settings → General → **Export Diagnostics…** writes a plain-text
  report (app/system info, display topology with color adjustments, open PIP windows, and
  recent log entries) to a location you choose — handy for bug reports. Nothing leaves
  your machine.

## 🔒 Security

- **Verified auto-updates.** Before installing, the updater downloads the release's
  published `SHA256SUMS.txt`, recomputes the SHA-256 of the downloaded `.app.zip`, and
  refuses to install if the hash is missing or doesn't match — closing a tampering /
  corruption vector on the privileged self-replace.

## 🔧 Changed

- The app bundle is now named **`Hydra Display.app`** — the menu bar, Dock, and Finder all
  show "Hydra Display". Download files keep the `HydraDisplay-x.y.z` prefix.
- Unified logging via `os.Logger` across display, update, and capture events.
- Removed the large "New Virtual Display" card from the Overview; use the **+** button in
  the toolbar (or the menu bar).

## ✅ Requirements

- **macOS 26.0 (Tahoe)** or later.
- Picture in Picture requires the **Screen Recording** permission (prompted on first use).

## ⬇️ Install

1. Download **`HydraDisplay.app.zip`** below and unzip it.
2. Move **`Hydra Display.app`** to `/Applications`.
3. The app is **not notarized** (it uses a private API and ships without a Developer
   ID), so Gatekeeper will warn on first launch. Either:
   - **Right-click → Open**, then confirm, **or**
   - run:
     ```bash
     xattr -dr com.apple.quarantine "/Applications/Hydra Display.app"
     ```

## ⚠️ Known limitations

- Creating displays relies on a **private Apple API** — it cannot ship on the Mac App
  Store and may break on a future macOS release. See
  [docs/PRIVATE_API.md](../PRIVATE_API.md).
- Localization currently covers English and Brazilian Portuguese only.
