# Hydra Display v0.2.0

**A big feature release.** Hydra Display gains real-time Picture in Picture, live
resolution switching, named profiles, Shortcuts support, persistence, in-app updates,
and a Brazilian Portuguese localization — still free, open source, and built for
macOS 26 (Tahoe).

## ✨ Highlights

- **Real-time Picture in Picture.** Open any display *or* a specific app window (e.g. a
  video player) in a floating, always-on-top window that streams live via
  ScreenCaptureKit — it stays visible even over full-screen apps.
- **Live resolution switching.** Change any display's active resolution (physical or
  virtual) from its Overview card or the detail pane, via the public CoreGraphics mode API.
- **Named profiles.** Save the current set of virtual displays as a named profile and
  re-apply it later in one click.
- **Shortcuts & Spotlight.** "Create Virtual Display" (with a resolution choice) and
  "Remove All Virtual Displays" App Intents, runnable from the Shortcuts app and Spotlight.
- **In-app auto-update (no Sparkle).** Checks GitHub Releases, downloads the new build,
  and installs it over the running app through the native macOS authentication dialog —
  no visible terminal.

## 📦 What's new

- **Persistence:** created virtual displays are saved and automatically recreated on the
  next launch (toggle in Settings).
- **Open at Login:** optional launch-at-login via ServiceManagement.
- **Global keyboard shortcuts** (opt-in): ⌃⌥⌘N creates a 4K Retina display, ⌃⌥⌘R removes all.
- **First-run onboarding** explaining the app and the private-API / Gatekeeper caveat.
- **Settings window** (⌘,) with the options above, plus a Settings shortcut in the menu bar.
- **Update-available indicators** in the sidebar and the menu bar.
- **Localization:** Brazilian Portuguese (pt-BR) for the core UI, via a String Catalog
  (English remains the source language).
- **Test suite:** `HydraDisplayTests` (Swift Testing) and `HydraDisplayUITests`, runnable with ⌘U.

## 🩹 Fixed

- The app now shows a consistent **"Hydra Display"** name in the menu bar, Dock, and
  Finder (via an explicit `Info.plist`), instead of the build target's "HydraDisplay".

## ✅ Requirements

- **macOS 26.0 (Tahoe)** or later.
- Picture in Picture requires the **Screen Recording** permission (prompted on first use).

## ⬇️ Install

1. Download **`HydraDisplay.app.zip`** below and unzip it.
2. Move **`Hydra Display.app`** to `/Applications`.
3. The app is **not notarized** (it uses a private API and ships without a Developer
   ID), so Gatekeeper will warn on first launch. Either:
   - **Right-click → Open**, then confirm, **or**
   - run:
     ```bash
     xattr -dr com.apple.quarantine "/Applications/Hydra Display.app"
     ```

## ⚠️ Known limitations

- Creating displays relies on a **private Apple API** — it cannot ship on the Mac App
  Store and may break on a future macOS release. See
  [docs/PRIVATE_API.md](../PRIVATE_API.md).
- Localization currently covers English and Brazilian Portuguese only.
