# Hydra Display v0.3.1

A focused fix release for Picture in Picture.

## 🩹 Fixed

- **Full-screen sources now appear in the PIP picker.** Windows on another Space — such
  as a full-screen video in Safari — and windows with no title are now listed. Background
  helper and menu-bar-agent apps are hidden so the list stays clean.
- **No more system stall on large windows.** PIP capture resolution is capped (longest
  side ≤ 2560 px), so picking a very large source like a full-screen Retina video no
  longer asks the GPU for a 5K+ 60 fps stream and freezes the machine.
- **Survives full-screen toggling.** A captured window that enters or leaves full screen
  (which gives it a new window ID) is re-bound automatically instead of going black.
- **Smoother under load.** PIP frames are coalesced — only the newest frame is drawn — so
  a burst of frames can't pile up on the main thread.

## ✅ Requirements

- **macOS 26.0 (Tahoe)** or later.
- Picture in Picture requires the **Screen Recording** permission (prompted on first use).

## ⬇️ Install

1. Download **`HydraDisplay.app.zip`** below and unzip it.
2. Move **`Hydra Display.app`** to `/Applications`.
3. The app is **not notarized** (it uses a private API and ships without a Developer
   ID), so Gatekeeper will warn on first launch. Either:
   - **Right-click → Open**, then confirm, **or**
   - run:
     ```bash
     xattr -dr com.apple.quarantine "/Applications/Hydra Display.app"
     ```
